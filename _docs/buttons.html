Default buttons

Default button is created with .button class applied to <a>, <button> or <input type=submit> tag.

Code

<a href="#" class="button">Default button</a>
<button class="button">Default button</button>
<input type="submit" class="button" value="Default button">

Button colors

Buttons can have as much color modes as you want - just set name/color pairs in $inputs_color_additional or global $color_additional map. Then use custom colors by adding color name suffix to button class.

Code

<button class="button">Primary</button>
<button class="button-success">Success</button>
<button class="button-error">Error</button>

Demo

Button styles

Buttons are implemented in two commonly used simple styles. Each style's class name is formed by adding a .button- prefix to the name of the type (except default flat).

Code

<p>
  <button class="button-outline">Primary</button>
  <button class="button-outline-success">Success</button>
  <button class="button-outline-error">Error</button>
</p>

Demo

Button groups

Buttons can be united into seamless group by wrapping them with .button-group element.

Code

<div class="button-group">
  <button class="button">Primary</button>
  <button class="button">Primary</button>
  <button class="button">Primary</button>
</div>
<div class="button-group">
  <button class="button">Primary</button>
  <button class="button-success">Success</button>
  <button class="button-error">Error</button>
</div>
<div class="button-group">
  <a href="#" class="button-outline">Primary</a>
  <a href="#" class="button-outline">Primary</a>
  <a href="#" class="button-outline">Primary</a>
</div>

Semantic classes / Mixins

If you are a fan of semantic and clean HTML code, you may like to use your own class names for structure blocks. We designed mixins that can apply framework styles to your own classes.

Mixins list

Mixin Description Variables 
ns_button_common()
 Add necessary for any button settings and style resets.
You shouldn't use it with ns_button-* mixins. It is included already		 $height, $padding, $font-size, $line-height, $border-width, $border-radius, $font-weight, $text-align, $background, $border-color, $color 
ns_button()
 Create flat button $height, $padding, $font-size, $line-height, $border-width, $border-radius, $font-weight, $text-align, $background, $border-color, $color 
ns_button-outline()
 Create outline button $height, $padding, $font-size, $line-height, $border-width, $border-radius, $font-weight, $text-align, $background, $border-color, $color

Examples:

SCSS

.custom-button-name {
  @include ns_button($height: 60px, $padding: 40px; $font-size: 20px; $background: #000); // create flat button
}

Setting buttons config

The module has common input settings and individual button settings. Settings are in _config.scss file.

Settings list

Name Description Values
$button_font-size Button font size px
$button_border-width Border width of default flat button px
$button_outline_border-width Border width of outline button px
$button_border-radius Border radius of buttons as in spec.
